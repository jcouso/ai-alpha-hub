### 6. Cursor 3.2: Agent Execution Runtime + Critical Vulnerability
Cursor 3.2 reframes the IDE as an "Agent Execution Runtime" — a significant architectural shift. But simultaneously, security researchers disclosed a critical vulnerability: Cursor's handling of hidden Git hooks allows arbitrary code execution. Opening a malicious repo in Cursor could RCE your machine. CSO Online and Hackread both covered it.

**Why it matters:** The architecture shift is interesting (IDE → agent runtime), but the vulnerability is urgent. If you use Cursor, check for updates immediately. This is the same class of supply-chain attack that hit VS Code extensions previously — but with an agent-powered IDE, the blast radius is larger because the agent has more system access.

## 🧑‍💻 Coding Tips

### 1. Try Codex on AWS Bedrock (if you're on AWS)
The OpenAI+AWS partnership just went live. You can now use Codex through Bedrock and count it against existing cloud credits:

```bash
# Check if you have access to the preview
aws bedrock list-foundation-models \
  --query "modelSummaries[?contains(providerName, 'OpenAI')]" \
  --output table

# Use OpenAI models through Bedrock (preview)
aws bedrock-runtime invoke-model \
  --model-id openai.gpt-5.5 \
  --content-type "application/json" \
  --body '{"messages":[{"role":"user","content":"Refactor this to use async iterators"}]}' \
  output.json
```

### 2. Test Mistral's Remote Agents via Vibe CLI
Mistral's cloud-based coding agents are a fundamentally different approach — no local compute needed:

```bash
# Install Vibe CLI
pip install mistral-vibe

# Launch a remote coding agent (runs in Mistral's cloud)
vibe agent run --task "Add rate limiting middleware to my Express API" \
  --model mistral-medium-3.5 \
  --repo .

# The agent executes remotely and pushes changes to a branch.
# Review diff, then merge. No local GPU needed.
```
