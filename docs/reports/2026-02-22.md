### 1. Boris Tane: "How I Use Claude Code" ‚Äî The Definitive Practitioner Playbook üî•üî•üî•
**Score: 9.5/10** | CONFIRMED | HN #1 at 847 points, 532 comments

Boris Tane published what is likely the most referenced Claude Code workflow post to date. After 9 months of daily use, his methodology:

**Five phases:**
1. **Research** ‚Äî Deep-read the codebase, write findings to `research.md`. Language matters: "deeply", "in great details", "intricacies" ‚Äî without these, Claude skims.
2. **Plan** ‚Äî Generate `plan.md` with approach, code snippets, file paths, trade-offs. Uses custom markdown files instead of Claude Code's built-in plan mode ("it sucks").
3. **Annotate** ‚Äî The developer's highest-leverage activity. Open `plan.md` in editor, add inline corrections. Send Claude back to update. Repeat 1-6 times until satisfied.
4. **Todo List** ‚Äî Convert approved plan into a numbered todo list in a new file.
5. **Implement** ‚Äî Only now does Claude write actual code. It follows the todo list item by item.

Key insight: *"The most expensive failure mode with AI-assisted coding isn't wrong syntax or bad logic ‚Äî it's implementations that work in isolation but break the surrounding system."* The research phase prevents this entirely.

**Why this matters for builders:** This is the workflow that's producing the best results at scale. The annotation cycle is the part most developers skip ‚Äî and it's where the human adds the most value. If you're doing prompt ‚Üí code ‚Üí fix errors, you're playing a different, worse game.

**Source:** [boristane.com/blog/how-i-use-claude-code](https://boristane.com/blog/how-i-use-claude-code/) | [HN discussion](https://news.ycombinator.com/item?id=43138241)

### 3. Llama 3.1 70B on a Single RTX 3090 via NVMe-to-GPU Bypass üíæüî•
**Score: 9.0/10** | CONFIRMED | HN Show at 356 points

A Show HN demonstrating Llama 3.1 70B running on a single RTX 3090 (24GB VRAM) by implementing direct NVMe-to-GPU data transfer that bypasses the CPU entirely for model weight loading. The technique leverages GPU-direct storage APIs to stream model weights from NVMe storage directly into GPU memory, dramatically reducing the memory bottleneck.

This matters because a 70B parameter model normally requires ~140GB of memory in FP16. By streaming weights from NVMe at the layer level, inference becomes possible on consumer hardware that wouldn't otherwise fit the model in memory.

**Why this matters for builders:** If you're running local inference for privacy or latency reasons (relevant for Claw setups), this expands what's possible on consumer GPUs. An RTX 3090 is ~$800 used. Running 70B models locally was previously a multi-GPU or cloud-only proposition.

### 10. Human Root of Trust ‚Äî Public Domain Agent Accountability Framework ‚öñÔ∏è
**Score: 7.0/10** | CONFIRMED | HN at 27 points

A public domain framework (v1.0, Feb 2026) for cryptographic accountability in autonomous agent systems. Core principle: **"Every agent must trace to a human."**

The problem statement: every digital system assumes a human is present on the other end. AI agents break this assumption ‚Äî they pass identity checks, execute transactions, sign contracts, and manage infrastructure autonomously. The framework proposes a six-step trust chain from action to authorization to human principal, using Ed25519 key pairs and verifiable delegation chains.

Dedicated to the public domain. No rights reserved. The authors explicitly state: "We are not the right people to finish this" ‚Äî it's an invitation for security engineers, cryptographers, and lawyers to build on.

**Why this matters:** As the Claw ecosystem grows and agents operate more autonomously, accountability becomes critical. This provides the conceptual framework (and early architecture) for answering "who is responsible for what this agent did?" ‚Äî the question regulators will inevitably ask.

Show HN: an MCP extension where agents communicate by meaning, not schema. **Practitioner alpha:** Worth watching as MCP evolves. The comparison table (MCP vs. Google A2A vs. Symplex) in the README shows where current protocols have gaps.

### Adopt Boris Tane's Five-Phase Claude Code Workflow
Stop doing prompt ‚Üí code ‚Üí fix errors. Start doing:
1. **Research**: `"read this folder in depth, understand how it works deeply‚Ä¶ write a detailed report in research.md"`
2. **Plan**: `"write a detailed plan.md outlining how to implement this. include code snippets"`
3. **Annotate**: Open plan.md in your editor, add inline corrections, send Claude back. Repeat 1-6x.
4. **Todo**: Convert plan into numbered todo list file
5. **Implement**: Now Claude writes code, following the todo list

The annotation cycle is where you add the most value. It's also where you prevent the most expensive failure: code that works in isolation but breaks the surrounding system.

**Source:** [boristane.com/blog/how-i-use-claude-code](https://boristane.com/blog/how-i-use-claude-code/)
