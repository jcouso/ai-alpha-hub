## 🧑‍💻 Coding Tips

### Study the Leaked Fable 5 System Prompt Architecture
The leaked Fable 5 system prompt (1,585 lines, 72 sections, 18 tool definitions) is now the most detailed public reference for how frontier models structure their operational behavior. Key patterns worth stealing for your own system prompts:

```markdown
# Key structural patterns from Fable 5's 120K-char system prompt:

# 1. Section-based organization (72 named sections)
# Each capability gets its own section with clear boundaries

# 2. Tool definitions as JSON schemas (18 tools)
# Each tool has explicit parameter schemas, not just descriptions

# 3. Thinking-effort calibration
# Different verbosity levels for different task complexity

# Your CLAUDE.md can borrow this structure:
## Section: Code Review
- When reviewing code, always check: [specific list]
- Output format: [structured template]
- Edge cases to watch: [explicit list]

## Section: Testing
- Default to red-green-refactor
- Always write the test FIRST, verify it fails, THEN implement
- Minimum coverage: [threshold]
```

**Source:** [GitHub: asgeirtj/system_prompts_leaks](https://raw.githubusercontent.com/asgeirtj/system_prompts_leaks/refs/heads/main/Anthropic/Claude%20Code/claude-code-2.1.172-fable-5.md)

### Use Addy Osmani's Agent Skills for Production-Grade Output
Instead of free-form agent prompting, install structured engineering skills:

```bash
# Clone Addy Osmani's production engineering skills
git clone https://github.com/addyosmani/agent-skills.git ~/.agent-skills

# Add to your CLAUDE.md or .cursorrules:
cat >> CLAUDE.md << 'EOF'

## Engineering Skills
- Follow PRD-first development: write the PRD before any code
- Use red-green-refactor: test → fail → implement → pass → refactor
- Feature flags for all new features (never ship behind a hard deploy)
- Thin slices: deliver the smallest working increment first
EOF
```
