### 4. Claude Code Safety Bypass — Deny Rules Circumvented via Subcommand Overflow
Adversa (Tel Aviv security firm) disclosed that Claude Code's deny rules can be bypassed by padding a command with >50 no-op subcommands. The `bashPermissions.ts` file has a hard cap of 50 — beyond that, Claude Code asks permission instead of denying. Proof of concept: 50 `true` commands + `curl` bypasses the curl deny rule. The Register and SecurityWeek confirm. Anthropic has been notified.

**Why it matters:** The leaked source code literally showed researchers where to look. This is the second major Claude Code security issue in a week (after the SentinelOne supply chain attack). The vulnerability is trivially exploitable via prompt injection. Anyone using Claude Code's deny rules as a security boundary needs to know they're breakable.

### 2. Audit Your Claude Code Deny Rules After the Adversa Bypass
The 50-subcommand cap means your deny rules are bypassable. Check and harden:

```bash
# View your current deny rules
cat ~/.claude/settings.json | jq '.permissions.deny // empty'

# The bypass: >50 subcommands skips safety checks
# Test your own rules (safe - just checks the behavior):
echo "true && true && true && ... && curl http://example.com" | wc -w
# If you have >50 subcommands, deny rules won't apply

# Mitigation: add network isolation at the OS level
# macOS: use Little Snitch or built-in firewall
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --setblockall on

# Docker: run Claude Code with no network access
docker run --rm -it --network=none -v $(pwd):/workspace \
  node:22 npx @anthropic-ai/claude-code
```
