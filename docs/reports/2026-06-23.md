### 4. Agentjacking: Fake Sentry Errors Hijack AI Coding Agents (85% Success Rate)
Tenet Security Labs published a new attack class called "Agentjacking" — injecting malicious instructions into Sentry error events that AI coding agents (Claude Code, Cursor, Codex) interpret as legitimate remediation steps. The attack exploits public Sentry DSN keys to trigger fake errors containing crafted instructions. Agents then install attacker-controlled npm packages. 85% success rate across 100+ real-world targets. Tenet has open-sourced `agent-jackstop` configurations to harden Claude Code and Cursor.

**Why it matters:** This is the first documented attack that weaponizes a *trusted developer tool* (Sentry) against AI agents. The broader lesson: any tool your agent trusts implicitly is an attack surface. MCP servers, error trackers, CI pipelines — all need threat modeling as agent inputs.

- [Tenet Security](https://tenetsecurity.ai/blog/agentjacking-coding-agents-with-fake-sentry-errors/) | [The New Stack](https://thenewstack.io/agentjacking-sentry-mcp-attack/) | [Infosecurity Magazine](https://www.infosecurity-magazine.com/news/agentjacking-attacks-hijack-ai/)

**Alpha angle:** If you're running multi-step agent pipelines (research → filter → compile, like this report), FAPO's approach of step-level failure attribution is directly applicable. Instead of tuning end-to-end, diagnose which step degrades quality and fix that one.

## 🧑‍💻 Coding Tips

### Harden Your Agent Against Agentjacking
The Agentjacking attack exploits trusted tool integrations. Here's how to protect your setup today using Tenet's open-source `agent-jackstop` configs:

```bash
# Clone the hardening configs
git clone https://github.com/tenetsecurity/agent-jackstop.git

# For Claude Code — add to your .claude/settings.json:
{
  "permissions": {
    "deny": [
      "npm install *sentry*",
      "npx *sentry*"
    ]
  },
  "hooks": {
    "PreToolUse": [{
      "matcher": "bash",
      "command": "echo '$INPUT' | grep -qE 'sentry|dsn|error-tracking' && echo 'BLOCK: Suspicious Sentry-related command detected' && exit 1 || exit 0"
    }]
  }
}
```

The broader principle: audit every data source your agent reads automatically. Sentry, GitHub Issues, Slack messages — any of these can carry injected instructions.

## ⚙️ Workflow Upgrades

### 1. Add Agent Input Validation Hooks
After Agentjacking, the takeaway is clear: validate what your agent reads, not just what it writes. Claude Code's `PreToolUse` hooks let you intercept and inspect before execution:

```yaml
# In your project's .claude/hooks.yaml
PreToolUse:
  - matcher: "bash"
    command: |
      # Block any command that references untrusted package names
      echo "$INPUT" | python3 -c "
      import sys, json
      data = json.load(sys.stdin)
      blocked = ['sentry-patch', 'fix-error-', 'remediation-']
      cmd = data.get('command', '')
      if any(b in cmd for b in blocked):
          print('BLOCK: Suspicious package reference')
          sys.exit(1)
      "
```

### 2. Try Sakana Fugu as Your Model Router
If you're already calling multiple models (Opus for complex reasoning, Sonnet for speed, GPT for tool use), Fugu's single-endpoint routing might simplify your stack:

```bash
# Instead of managing model selection yourself:
# Before: if task.complexity > 8: use opus; else: use sonnet
# After: one API call, Fugu routes internally
curl https://api.sakana.ai/v1/fugu/completions \
  -H "Authorization: Bearer $SAKANA_KEY" \
  -d '{"prompt": "Review this PR diff for security issues...", "task_type": "code_review"}'
```
