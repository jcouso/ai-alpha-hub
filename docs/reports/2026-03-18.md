### 4) Meta REA — Autonomous ML Agent That 2x'd Model Accuracy
Meta published the full architecture of their Ranking Engineer Agent (REA) on March 17. REA autonomously generates hypotheses, launches training jobs, debugs failures, and iterates — running asynchronous workflows spanning days to weeks via a hibernate-and-wake mechanism. Production results: 2x model accuracy improvement, 5x engineering output (3 engineers + REA did work that previously needed 16 engineers across 8 models).

**Why it matters:** This is the most detailed public documentation of a production autonomous ML agent at scale. The hibernate-and-wake pattern is the key architectural insight — agents that can sleep between async operations (training runs, CI jobs) and resume with full context. If you're building agent workflows that span hours or days, study this architecture.

### 5) CursorJack — MCP Deeplinks as Attack Vector in Cursor IDE
Proofpoint disclosed "CursorJack," a proof-of-concept showing how Cursor IDE's cursor:// MCP deeplinks can be abused via social engineering to install malicious MCP servers or execute arbitrary commands. The attack requires user interaction (clicking a crafted link), but the risk is real given how many developers install MCP servers from untrusted sources.

**Why it matters:** This is the first named attack specifically targeting MCP in coding IDEs. As MCP becomes the standard protocol for connecting AI agents to tools, the attack surface grows. Expect similar disclosures for Windsurf, Claude Code, and other MCP-enabled environments. Practical defense: audit your MCP server allowlist, never click cursor:// links from untrusted sources.

## 🧪 Research Frontier

### Meta's Hibernate-and-Wake Pattern for Long-Running Agents
The most interesting technical detail in Meta's REA paper is the hibernate-and-wake mechanism for multi-day agent workflows. REA doesn't stay running during training jobs — it serializes its state, hibernates, and wakes when results are ready. This solves the fundamental problem with long-running agents: you can't keep a context window open for days.

**Alpha angle:** If you're building agent workflows that involve async operations (CI/CD, deployment, data processing), the hibernate-and-wake pattern is the architecture to study. Serialize state → sleep → wake on event → resume with full context. This is how agents scale beyond single-session interactions.

## 🧑‍💻 Coding Tips

### 1) Use GPT-5.4 Nano for Batch Processing
The new nano model is absurdly cheap for structured data tasks. Here's how to batch-describe images using Simon Willison's `llm` tool:

```bash
# Install/update llm with OpenAI support
pip install -U llm

# Describe a single image (costs ~0.07 cents)
llm -m gpt-5.4-nano -a photo.jpg 'describe this image in 2 sentences'

# Batch process a folder of images
for f in ./images/*.jpg; do
  echo "=== $f ===" >> descriptions.txt
  llm -m gpt-5.4-nano -a "$f" 'describe image' >> descriptions.txt
done
```

### 2) Audit Your MCP Server Configs After CursorJack
Given the CursorJack disclosure, check what MCP servers you have installed and remove any from untrusted sources:

```bash
# Check Cursor MCP configs
cat ~/.cursor/mcp.json 2>/dev/null | jq '.mcpServers | keys'

# Check Claude Code MCP configs
cat ~/.claude/mcp_settings.json 2>/dev/null | jq '.mcpServers | keys'

# If you see anything you don't recognize, remove it
# Never click cursor:// or vscode:// links from untrusted sources
```
