### 2. GitHub Agentic Workflows ‚Äî CI/CD in Markdown
**Dropped:** Feb 13 | **Source:** [GitHub Changelog](https://github.blog/changelog/2026-02-13-github-agentic-workflows-are-now-in-technical-preview/) ¬∑ [GitHub Blog](https://github.blog/ai-and-ml/automate-repository-tasks-with-github-agentic-workflows/) ¬∑ [The New Stack](https://thenewstack.io/github-agentic-workflows-overview/)

Write automations in plain Markdown, drop them in `.github/workflows/`, and the `gh aw` CLI compiles them into standard GitHub Actions. AI agents handle issue triage, PR reviews, CI failure analysis. Read-only by default, preapproved "safe outputs" for writes. Open source.

**Why it matters:** Natural language CI/CD is a paradigm shift. No more YAML spaghetti. PRs still require human review. This is "Continuous AI" ‚Äî judgment-based automation layered on top of rule-based CI.

**Try it:**
```bash
gh extension install github/gh-aw
# Create a .md file in .github/workflows/ describing what you want
gh aw compile
```

### 3. MiniMax M2.5 ‚Äî Open-Weight Sonnet-Tier at 1/20th Cost
**Dropped:** Feb 12 | **Source:** [VentureBeat](https://venturebeat.com/technology/minimaxs-new-open-m2-5-and-m2-5-lightning-near-state-of-the-art-while) ¬∑ [OpenHands Blog](https://openhands.dev/blog/minimax-m2-5-open-weights-models-catch-up-to-claude) ¬∑ [HuggingFace](https://huggingface.co/MiniMax)

230B parameter model, MIT license, MoE architecture (few active params = efficient). Lightning variant for speed. Cline added free promo integration same day (v3.59). OpenHands confirmed it matches Sonnet-class quality on their benchmarks.

**Why it matters:** First truly competitive open-weight model for agentic coding workloads. Self-host for ~$0.25/$1.25 per 1M tokens vs Opus at $5/$25. Great for high-volume draft generation, CI bots, or cost-sensitive agent loops.

### 4. OpenAI Codex Rust Rewrite ‚Äî Structured Network Approvals
**Dropped:** Feb 12-14 (6+ releases) | **Source:** [GitHub Releases](https://github.com/openai/codex/releases) ¬∑ [Commits](https://github.com/openai/codex/commits/main)

The Codex CLI Rust rewrite is on fire. Key this week: structured network approval plumbing (agents now request specific host/protocol access instead of blanket network), permissions history tracking in TUI, model-switch compaction handling. The MCP commands also dropped the "[experimental]" tag.

**Why it matters:** The new permission model is exactly what you need to safely run coding agents in prod. Network access is no longer all-or-nothing ‚Äî agents declare exactly what hosts they need. This is the sandbox model everyone's been asking for.

### 5. CloudRouter ‚Äî VMs/GPUs as a Claude Code Skill
**Dropped:** Feb 13 | **Source:** [cloudrouter.dev](https://cloudrouter.dev/) ¬∑ [HN (122pts)](https://news.ycombinator.com/item?id=47006393)

Install as a skill with `npx skills add manaflow-ai/cloudrouter`. Creates cloud sandboxes (E2B or Modal), run commands, transfer files, automate browsers via CDP ‚Äî all from Claude Code or Codex. Supports VNC desktop, VS Code in browser, pty.

**Why it matters:** Solves the "agent shares my machine" problem. Run multiple agents in parallel, each in its own sandbox. GPU access means ML tasks can spin up on demand. Perfect for teams running agent farms.
