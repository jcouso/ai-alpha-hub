## 🧑‍💻 Coding Tips

### Autoresearch Loop Setup (steal Karpathy's pattern)

The autoresearch pattern is dead simple. One file, one metric, one loop. Here's the minimal setup:

```markdown
# program.md (system prompt for your agent)

## Goal
Improve the eval metric in train.py by any means necessary.

## Rules
- Only modify train.py
- After each edit: run `python train.py`, then `python eval.py`
- If metric improves: git commit with description of what changed
- If metric worsens or errors: git revert, try something different
- Document your reasoning in scratchpad.md before each attempt

## Phases
1. Hyperparameter tuning (learning rate, batch size, schedules)
2. Small architectural changes (attention heads, layer depth)  
3. Moonshot ideas (new loss functions, novel architectures)
4. Unconstrained (web search for papers, try anything)
```

Run it:
```bash
# In your project dir with train.py and eval.py
claude-code --system-prompt program.md --auto-approve
# Or with Codex:
codex "Follow the instructions in program.md exactly. Start phase 1."
```
