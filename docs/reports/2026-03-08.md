## 🧑‍💻 Coding Tips

### 1) Sandbox your AI agents from production infrastructure
After the Claude Code database deletion incident — here's a minimal safety setup:

```bash
# Create a restricted IAM policy for your agent's AWS credentials
cat > agent-policy.json << 'EOF'
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:PutObject",
        "ec2:DescribeInstances"
      ],
      "Resource": "*"
    },
    {
      "Effect": "Deny",
      "Action": [
        "rds:DeleteDBInstance",
        "rds:DeleteDBSnapshot",
        "ec2:TerminateInstances",
        "s3:DeleteBucket",
        "iam:*"
      ],
      "Resource": "*"
    }
  ]
}
EOF

# Apply to your agent's role
aws iam put-role-policy \
  --role-name ai-agent-role \
  --policy-name deny-destructive \
  --policy-document file://agent-policy.json
```

Rule of thumb: agents get read + create permissions. Never delete or terminate. If the agent needs to destroy something, it should output the command for human execution.
