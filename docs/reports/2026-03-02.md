### 2) Hackers weaponize Claude Code to breach 10 Mexican government agencies ‚Äî 150GB stolen
- Gambit Security disclosed a cyberattack where hackers used Claude Code and ChatGPT to develop exploits, create custom tools, and automatically exfiltrate 150GB+ of data from 10 Mexican government bodies and a financial institution. Data includes 195 million taxpayer records, voting records, and government employee databases. The attack chain started with Mexico's tax authority in late December and expanded using AI-generated tooling.
- Why it matters: first confirmed large-scale government breach where AI coding agents were the primary offensive tool. The attacker jailbroke Claude to write exploits and automate data extraction ‚Äî exactly the workflow practitioners use for legitimate work, repurposed for offense.
- Sources: [SecurityWeek](https://www.securityweek.com/hackers-weaponize-claude-code-in-mexican-government-cyberattack/), [Security Affairs](https://securityaffairs.com/188696/ai/claude-code-abused-to-steal-150gb-in-cyberattack-on-mexican-agencies.html), [Security Boulevard](https://securityboulevard.com/2026/03/hacker-uses-claude-chatgpt-ai-chatbots-to-breach-mexican-government-systems/)

### 3) ClawJacked: OpenClaw WebSocket hijack + 71 malicious ClawHub skills
- Oasis Security disclosed "ClawJacked" ‚Äî a vulnerability where malicious websites could open WebSocket connections to OpenClaw's localhost gateway (TCP 18789) and brute-force the gateway password. Separately, 71 malicious skills on ClawHub were found distributing Atomic Stealer (macOS info-stealer). OpenClaw also patched a log poisoning vulnerability. All issues are now fixed.
- Why it matters: **if you run OpenClaw, update immediately.** The WebSocket attack could be triggered by simply visiting a malicious website. The ClawHub supply chain attack mirrors npm/PyPI malware campaigns ‚Äî agent skill marketplaces are the new attack surface.
- Sources: [The Hacker News](https://thehackernews.com/2026/02/clawjacked-flaw-lets-malicious-sites.html), [BleepingComputer](https://www.bleepingcomputer.com/news/security/clawjacked-attack-let-malicious-websites-hijack-openclaw-to-steal-data/)

## üßë‚Äçüíª Coding Tips

### 1) Check if your OpenClaw is patched against ClawJacked
The ClawJacked vulnerability lets any malicious website connect to your local OpenClaw gateway. Check your version and update:

```bash
# Check your OpenClaw version
openclaw --version

# Update to latest (includes ClawJacked + log poisoning fixes)
npm update -g openclaw

# Verify gateway isn't exposed beyond localhost
lsof -i :18789 | grep LISTEN

# Check for suspicious WebSocket connections from browser processes
lsof -i :18789 | grep -E '(Chrome|Firefox|Safari|Arc)'

# Audit installed skills for known malicious patterns
find ~/.openclaw/skills -name "*.md" -exec grep -l 'curl.*91\.92\.242' {} +
find ~/.openclaw/skills -name "*.sh" -exec grep -l 'openclawcli\.vercel' {} +
```

### 2) Sandbox AI-generated code before execution with OpenSandbox
Given today's Mexico breach and security stories, sandbox untrusted code execution:

```bash
# Quick start with Alibaba OpenSandbox (Docker)
docker run -d --name opensandbox \
  -p 8080:8080 \
  -v /var/run/docker.sock:/var/run/docker.sock \
  alibaba/opensandbox:latest

# Python SDK ‚Äî execute AI-generated code in isolation
pip install opensandbox-sdk

# Example: sandbox a Claude Code output before running locally
from opensandbox import SandboxClient

client = SandboxClient("http://localhost:8080")
sandbox = client.create(image="python:3.12-slim", timeout=30)

result = sandbox.execute("""
# paste AI-generated code here for safe execution
import os
print(os.getcwd())  # runs in sandbox, not your machine
""")
print(result.stdout)
sandbox.destroy()
```
