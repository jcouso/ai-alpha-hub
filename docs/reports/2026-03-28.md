## 🧪 Research Frontier

### IndexCache: 1.82x Faster Inference on Long-Context Models
Researchers at Tsinghua University and Z.ai built IndexCache, a sparse attention optimizer that cuts up to 75% of redundant computation. Results: 1.82x faster time-to-first-token and 1.48x faster generation throughput on long-context workloads.

**Alpha angle:** If you're running local models with long contexts (128K+ tokens), this technique could dramatically reduce inference costs. It targets the exact bottleneck that makes long-context agent loops expensive.
- VentureBeat: https://venturebeat.com/technology/indexcache-a-new-sparse-attention-optimizer-delivers-1-82x-faster-inference

### Google TurboQuant: 6x KV Cache Compression, Zero Accuracy Loss
Google introduced TurboQuant, a compression algorithm that reduces LLM key-value cache memory by 6x and delivers up to 8x speedup with no accuracy degradation. Works on existing models without retraining.

**Alpha angle:** KV cache is the main memory bottleneck for long conversations and agent loops. 6x compression means you can serve 6x more concurrent users on the same hardware, or run much longer contexts on consumer GPUs. This is production-ready infrastructure.
- MarkTechPost: https://www.marktechpost.com/2026/03/27/google-introduces-turboquant/

### jsongrep: DFA-Based JSON Querying (faster than jq)

```bash
# New tool trending on HN (377 pts) — orders of magnitude faster than jq for path-based queries
cargo install jsongrep

# Find all "name" fields in a large JSON file
jsongrep '$.*.name' large-dataset.json

# Pipe with ripgrep for filtered searches
jsongrep '$.users.*' data.json | rg '"active": true'

# Not a full jq replacement (no filters/arithmetic), but blazing fast for path-based extraction
```
