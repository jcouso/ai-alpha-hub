### 3) NVIDIA Drops Nemotron 3 Super — 120B Open-Weight Agentic Model
NVIDIA released Nemotron 3 Super, a 120B total / 12B active parameter MoE model using a hybrid Mamba-Transformer architecture with LatentMoE. Open weights, open data, open recipe. 5x higher throughput than GPT-OSS-120B in FP4. 1M-token context window. Scores 85.6% on PinchBench (agent benchmark). "Ultra is coming."

**Why this matters:** This is the most capable open model specifically designed for agentic workloads. The Mamba-Transformer hybrid + LatentMoE architecture is architecturally novel — not just another transformer scale-up. For anyone running AI agents (OpenClaw, Claude Code, custom): this could be your local/self-hosted brain at a fraction of API costs. The 1M context window makes it viable for real codebases. Pre-GTC flex from NVIDIA.

- Sources: [NVIDIA Blog](https://developer.nvidia.com/blog/introducing-nemotron-3-super-an-open-hybrid-mamba-transformer-moe-for-agentic-reasoning/), [NVIDIA Research](https://research.nvidia.com/labs/nemotron/Nemotron-3-Super/), [VideoCardz](https://videocardz.com/newz/nvidia-releases-nemotron-3-super-120b-open-model-with-1m-token-context)

## 🧪 Research Frontier

### METR: Many SWE-bench-Passing PRs Would Not Be Merged (248 pts on HN)
METR researchers found that many AI-generated PRs that "pass" SWE-bench would never be merged by real maintainers. The study questions the validity of SWE-bench as a proxy for actual software engineering capability. Trending heavily on Hacker News.

**Alpha angle:** If you're evaluating coding agents by SWE-bench scores alone, you're measuring the wrong thing. The gap between "passes tests" and "mergeable code" is where real engineering value lives. This validates the need for AI code review tools (Claude Code Review, Copilot CCR) as a complement to AI code generation.

- Sources: [METR](https://metr.org/notes/2026-03-10-many-swe-bench-passing-prs-would-not-be-merged-into-main/), [HN — 248 pts](https://news.ycombinator.com/item?id=47323017)

### Microsoft BitNet Trending on HN (349 pts) — 1-Bit LLM Inference on Single CPU
Microsoft's BitNet framework, which enables 100B parameter LLM inference on a single CPU using 1.58-bit ternary weights, is trending heavily. Up to 6x faster inference and 82% lower energy consumption. The architecture uses {-1, 0, 1} weights instead of floating point.

**Alpha angle:** This is the "local AI" play. If 100B models can run on commodity CPUs at 5-7 tokens/sec, the entire inference cost structure changes. Combined with Nemotron 3 Super's Mamba-Transformer hybrid, we're seeing two parallel paths to affordable AI: better architectures (Mamba) and better quantization (1-bit). Both reduce dependence on expensive GPU inference.

- Sources: [GitHub — microsoft/BitNet](https://github.com/microsoft/BitNet), [HN — 349 pts](https://news.ycombinator.com/)

## 🧑‍💻 Coding Tips

### 1) Copilot Code Review from Terminal
GitHub just added CLI support for Copilot code review. Add it to your workflow:

```bash
# Request Copilot review on PR creation
gh pr create --reviewer copilot

# Add Copilot review to existing PR
gh pr edit 123 --add-reviewer copilot
```

Combine with the agentic architecture upgrade (March 5) — Copilot now does multi-step analysis with tool calling, not just line-by-line comments.

### 2) Try Gemini CLI's Plan Mode for Safe Codebase Analysis
Gemini CLI's new Plan Mode analyzes your request in read-only mode before making changes:

```bash
gemini --plan "Refactor the auth module to use JWT tokens"
```

It reads your codebase, plans the changes, and lets you review before executing. Perfect for large refactors where you want to understand the blast radius first.

## ⚙️ Workflow Upgrades
1. **If you use Copilot, start reviewing from CLI.** The `--reviewer copilot` flag on `gh pr create` is the fastest path to AI-assisted code review without leaving your terminal. Combine with your existing human review process — Copilot reviews don't count toward required approvals.

2. **Evaluate Nemotron 3 Super for self-hosted agent workloads.** If you're paying for API calls to run coding agents, the 120B MoE with 1M context might be your escape hatch. Check the [model card](https://research.nvidia.com/labs/nemotron/Nemotron-3-Super/) and test on your specific use case before committing.

3. **Read OpenAI's prompt injection defense guide** before shipping any agentic feature. The patterns (input validation, tool permission scoping, output filtering) are applicable regardless of which model you use.
