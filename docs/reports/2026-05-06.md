## 🧪 Research Frontier

### ProgramBench: Can AI Rebuild Programs From Scratch?
Facebook Research released ProgramBench — a benchmark that asks AI coding agents to rebuild complete programs from only compiled binaries and documentation. Results: Claude Opus 4.7 leads but scores are low across all models. The benchmark exposed that current agentic coding systems fail at the architectural planning required for large binary reconstruction. Active discussion on r/LocalLLaMA about whether multi-agent approaches could push scores higher.

**Alpha angle:** This is the benchmark that separates "coding assistant" from "software engineer." If you're evaluating AI coding tools for production use, ProgramBench scores are a better signal than SWE-Bench for complex codebases. Also: `pip install programbench` — you can run it yourself.

- Sources: [GitHub (facebookresearch/programbench)](https://github.com/facebookresearch/programbench) · [BenchLM](https://benchlm.ai/benchmarks/programBench) · [r/LocalLLaMA](https://www.reddit.com/r/LocalLLaMA/comments/1t4j4s9/)

### Gemma 4 Multi-Token Prediction Drafters
Google released MTP (Multi-Token Prediction) drafters for the Gemma 4 family — a speculative decoding technique that achieves up to 3x inference speedup. Instead of generating one token at a time, MTP drafters predict multiple future tokens in parallel and verify them. The drafters are available as separate models on Hugging Face and work with vLLM.

**Alpha angle:** This makes self-hosted Gemma 4 dramatically more practical for production workloads. The technique is model-agnostic in principle — expect other open models to adopt MTP architectures. If you're serving Gemma 4 via vLLM, the TurboQuant fix for Qwen 3.5 (also merged this week) suggests the serving stack is maturing fast.

- Sources: [Google Blog](https://blog.google/innovation-and-ai/technology/developers-tools/multi-token-prediction-gemma-4/) · [AIToolly](https://aitoolly.com/ai-news/article/2026-05-06-google-boosts-gemma-4-performance-multi-token-prediction-drafters-deliver-3x-faster-inference)

### AIMap: 1M+ Exposed AI Services Found
Bishop Fox released AIMap, an open-source tool that scans for exposed AI endpoints at internet scale — MCP servers, Ollama instances, inference APIs. Their initial scan found roughly 1 million exposed AI services across 2 million hosts, many without authentication or safety controls.

**Alpha angle:** If you're running any AI services (MCP servers, local Ollama, custom APIs), run AIMap against your own infrastructure now. The attack surface for AI services is vastly underestimated. `aimap` is open-source and protocol-aware — it tests MCP, OpenAI-compatible endpoints, and more.

- Sources: [Help Net Security](https://www.helpnetsecurity.com/2026/05/06/aimap-ai-attack-surface-discovery/) · [Bishop Fox](https://bishopfox.com/tools/aimap)

### NVIDIA: Co-Design Patterns for Agentic System Complexity
NVIDIA's technical blog published a deep dive on "Extreme Co-Design" for agentic systems — arguing that as agent complexity rises, hardware-software co-design becomes critical. The piece covers inference chip architectures optimized for agent workflows, memory hierarchies for long-context agents, and KV cache management strategies.

**Alpha angle:** If you're building multi-agent systems that hit latency walls, this is required reading. The inference chip discussion aligns with YC's "Inference Chips for Agent Workflows" episode from this week.

## 🧑‍💻 Coding Tips

### Try GPT-5.5 Instant via API Today

The model is live on the API as `gpt-5.5-2026-04-23`. Quick test against your current workflow:

```bash
# Compare GPT-5.5 Instant vs your current model
curl https://api.openai.com/v1/chat/completions \
  -H "Authorization: Bearer $OPENAI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-5.5-2026-04-23",
    "messages": [{"role": "user", "content": "Write a TypeScript function that validates and sanitizes MCP tool inputs with zod schemas. Include error handling."}],
    "temperature": 0
  }'
```

### Scan Your AI Endpoints with AIMap

If you're running any AI services (Ollama, MCP servers, custom APIs), check your exposure:

```bash
# Install and scan your own infrastructure
pip install aimap
aimap scan --target 192.168.1.0/24 --protocols mcp,openai,ollama
aimap report --format json > ai-exposure-report.json
```

Bishop Fox found 1M+ exposed services. Don't be one of them.
