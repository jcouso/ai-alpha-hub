## 🧑‍💻 Coding Tips

### Try GPT-5.6's new "ultra" sub-agent mode via API

When GPT-5.6 becomes available to you, the new `ultra` reasoning effort spawns sub-agents internally:

```bash
# GPT-5.6 Sol with ultra mode (sub-agent orchestration)
curl https://api.openai.com/v1/responses \
  -H "Authorization: Bearer $OPENAI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-5.6-sol",
    "reasoning": { "effort": "ultra" },
    "input": "Refactor this codebase to use dependency injection throughout. Review each file, create the DI container, and verify all tests pass.",
    "tools": [{ "type": "code_interpreter" }]
  }'
```

### Audit your MCP configs after the Amazon Q CVE

The Amazon Q exploit worked via `.amazonq/mcp.json` in repos. Check YOUR projects for similar attack vectors:

```bash
# Find all MCP config files in your workspace
find ~/code -name "mcp.json" -o -name "mcp.yaml" -o -name ".mcp" | head -20

# Check for any MCP server URLs pointing to unknown hosts
grep -r "url.*http" ~/.cursor/mcp.json ~/.config/claude-code/mcp.json 2>/dev/null

# Verify no repo has injected MCP configs
for repo in ~/code/*/; do
  if [ -f "$repo/.amazonq/mcp.json" ] || [ -f "$repo/.cursor/mcp.json" ]; then
    echo "⚠️  MCP config found in repo: $repo"
  fi
done
```

## ⚙️ Workflow Upgrades

### 1. Set up cost tracking per coding task
With the "tokenmaxxing → efficiency" shift, start measuring what you spend:

```bash
# Add to your CLAUDE.md / .cursorrules
# COST AWARENESS: Before starting any task, estimate token count.
# After completion, log: task, model used, input/output tokens, wall time.
# Target: <$2 per feature, <$0.50 per bug fix.
```

### 2. Add MCP config to .gitignore globally
After the Amazon Q CVE, prevent MCP configs from being committed to repos:

```bash
# Add to ~/.gitignore_global
echo ".amazonq/" >> ~/.gitignore_global
echo ".cursor/mcp.json" >> ~/.gitignore_global
git config --global core.excludesfile ~/.gitignore_global
```
