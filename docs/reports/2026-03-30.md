## 🧑‍💻 Coding Tips

### 1. Set up Claude Code auto mode (shipped last week)
Stop hitting "y" on every permission prompt. Auto mode lets Claude Code self-decide permissions with built-in safety rails — a middle ground between default (interrupt everything) and YOLO mode:

```bash
# Enable auto mode in your project
claude config set autoMode true

# Or toggle per-session with the flag
claude --auto

# Auto mode uses a safety classifier to decide:
# ✅ Auto-approve: file reads, writes within project, test runs
# ⚠️  Still prompts: destructive ops, network requests, system commands
# 🚫 Always blocks: rm -rf, credential access, production deploys
```

### 2. Connect Codex to your tools via MCP plugins
OpenAI's new plugin system lets Codex connect to external tools through MCP servers:

```bash
# Install a Codex plugin (e.g., connect to your dev environment)
codex plugin install @openai/mcp-sandbox

# Create a plugin config (.codex/plugins.json)
cat > .codex/plugins.json << 'EOF'
{
  "plugins": [
    {
      "name": "sandbox",
      "type": "mcp",
      "config": {
        "middleware": ["node", "python3"],
        "env": { "NODE_ENV": "development" }
      }
    }
  ]
}
EOF

# Codex now has sandbox access through MCP — test it
codex "set up a dev environment and run the test suite"
```
