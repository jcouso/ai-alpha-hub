## 🧑‍💻 Coding Tips

### Try Opus 4.8 Effort Controls in Claude Code

Opus 4.8's effort levels let you trade cost for depth. In Claude Code:

```bash
# Set effort level for current session
claude config set effort extra

# Or per-command with the API
curl https://api.anthropic.com/v1/messages \
  -H "x-api-key: $ANTHROPIC_API_KEY" \
  -H "anthropic-version: 2024-06-01" \
  -H "content-type: application/json" \
  -d '{
    "model": "claude-opus-4-8-20260529",
    "max_tokens": 8192,
    "thinking": {"type": "enabled", "budget_tokens": 4096},
    "messages": [{"role": "user", "content": "Refactor this module..."}]
  }'
```

Use `high` (default) for routine tasks, `extra` for complex refactors, `max` for codebase-scale migrations. The cost difference is meaningful at scale — especially after that $500M cautionary tale.

### Quick SQLite Durable Workflow Pattern

Inspired by the trending Obelisk approach (570 pts on HN):

```python
import sqlite3, json, time

def durable_step(db, workflow_id, step_name, fn):
    """Execute a workflow step with SQLite-backed durability."""
    cur = db.execute(
        "SELECT result FROM steps WHERE workflow_id=? AND name=?",
        (workflow_id, step_name)
    )
    row = cur.fetchone()
    if row:
        return json.loads(row[0])  # replay from history

    result = fn()  # execute
    db.execute(
        "INSERT INTO steps (workflow_id, name, result, ts) VALUES (?,?,?,?)",
        (workflow_id, step_name, json.dumps(result), time.time())
    )
    db.commit()
    return result
```

No Temporal, no Redis, no Postgres. Just SQLite. Works great for agent orchestration pipelines.
