### 5. Cursor CVE-2026-26268: Critical RCE via Git Hooks (CVSS 9.9)
A critical vulnerability in Cursor IDE (versions before 2.5) allows remote code execution through hidden Git hooks. When Cursor's AI agent autonomously runs `git checkout` or `git commit`, a malicious pre-commit hook embedded in a repository executes silently — outside the agent's reasoning chain and outside the user's view. CVSS 9.9. Fix: update to Cursor 2.5+.

**Why it matters:** This is the security story that makes the Uber budget crisis look minor. When your AI coding agent autonomously executes git operations, it inherits every trust assumption in the repository — including malicious hooks. Update immediately, and start auditing repos for suspicious hooks before cloning.

- Sources: [CSO Online](https://www.csoonline.com/article/4164250/critical-cursor-bug-could-turn-routine-git-into-rce.html) · [Hackread](https://hackread.com/cursor-ai-ide-vulnerability-code-execution-git-hooks/) · [NVD](https://novee.security/blog/cursor-ide-cve-2026-26268-git-hook-arbitrary-code-execution/)

```bash

# Scan repos for suspicious git hooks
find ~/code -path "*/.git/hooks/*" -type f ! -name "*.sample" \
  -exec echo "⚠️  Active hook: {}" \;

# Check for nested bare repos (attack vector)
```
