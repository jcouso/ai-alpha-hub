## üßë‚Äçüíª Coding Tips

### 1) Scan repos for malicious agent config files before running coding agents
The Claude Code CVEs showed that `.claude/settings.json`, Hooks, and MCP configs can execute arbitrary shell commands. Add this to your pre-clone workflow:

```bash
#!/usr/bin/env bash
# agent-config-audit.sh ‚Äî scan a repo for risky agent config patterns
set -euo pipefail

REPO_DIR="${1:-.}"
RISKY_PATTERNS='(curl |wget |bash -c|python -c|rm -rf|eval |nc |/dev/tcp|base64 -d)'

echo "üîç Scanning $REPO_DIR for risky agent config files..."

find "$REPO_DIR" -type f \( \
  -path "*/.claude/*" \
  -o -name "AGENTS.md" \
  -o -name "CLAUDE.md" \
  -o -name ".mcp.json" \
  -o -name "mcp_config.json" \
  -o -path "*/.cursor/*" \
  -o -name "SKILL.md" \
\) -exec grep -lEn "$RISKY_PATTERNS" {} + 2>/dev/null && {
  echo "‚ö†Ô∏è  RISKY PATTERNS FOUND ‚Äî review before running any coding agent"
  exit 1
} || echo "‚úÖ No risky patterns detected"
```

### 2) Audit your Google Cloud projects for Gemini API exposure
If you've ever enabled the Generative Language API, your existing API keys may now authenticate to Gemini:

```bash
# List all API keys in your GCP project and check for Gemini API enablement
gcloud services list --enabled --project YOUR_PROJECT_ID | grep -i generativelanguage

# If enabled, list and rotate any public-facing API keys
gcloud services api-keys list --project YOUR_PROJECT_ID \
  --format='table(uid,displayName,restrictions.apiTargets)'

# Restrict keys to only the services they need
gcloud services api-keys update KEY_ID \
  --api-target=service=maps-backend.googleapis.com
```
