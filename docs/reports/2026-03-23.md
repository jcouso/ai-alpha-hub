## 🔥 Lead Stories

### 1. Cursor Admits Composer 2 Built on Kimi K2.5 — Attribution Firestorm Escalates
TechCrunch confirmed: Cursor's new Composer 2 coding model is built on top of Moonshot AI's Kimi K2.5 (Chinese open-source model). After community sleuths spotted references to Kimi in internal identifiers, co-founder Aman Sanger admitted the base, stating "only ~1/4 of compute came from the base." Elon Musk weighed in ("Yeah, it's Kimi 2.5"). Moonshot's Kimi account confirmed it was "an authorized commercial partnership" via Fireworks AI. The model still beats Opus 4.6 on SWE-bench — but the transparency hit is real. Cursor was already under pressure at a ~$30B valuation; this is the kind of trust erosion that compounds.

**Why it matters for builders:** Model provenance is now a brand risk. If you're building on open-weight models and charging a premium, transparent attribution is table stakes. The "fine-tune and rebrand" playbook just got harder.

### 2. China Issues Official OpenClaw Security Guidance
China's National Internet Emergency Center (CNCERT) and Cybersecurity Association jointly released government-level security guidance for OpenClaw. Recommendations: install on dedicated devices/VMs/containers, maintain strict environment isolation, avoid running on everyday work machines. They flagged risks in architectural design, default settings, vulnerability management, plugin ecosystem, and behavioral control. This was published by Xinhua, People's Daily, and CGTN within hours. Separately, a Composio blog post "OpenClaw is a security nightmare dressed up as a daydream" hit #27 on HN with 354 points and 236 comments, and DEV Community published an "8-step hardening guide" noting 135,000+ instances running unprotected on the public internet.

**Why it matters for builders:** OpenClaw security is now a sovereign-level concern, not just a hacker hobby. If you ship OpenClaw integrations, expect enterprise buyers to demand hardening docs. The 135K naked instances stat is ammunition for every sales call from security vendors.

Core thesis: AI changes what code looks like, not whether code exists. ### 5. CursorJack Vulnerability (CVE-2026-31854) — AI IDE Security Gets Real
Security researchers at Wiz disclosed a code execution vulnerability in Cursor's development environment, dubbed "CursorJack." The attack path allows exploitation through the AI integration layer. Infosecurity Magazine covered it separately. This comes as Cursor is simultaneously dealing with the Kimi attribution controversy — a rough week for the $30B startup.

**Why it matters for builders:** AI coding tools run with your local file system permissions. A vulnerability here isn't a "data breach" — it's arbitrary code execution on developer machines. Audit your AI tool security posture, especially if agents have filesystem/terminal access.
