## 🚀 Top Moves (last 24h)

### 1) Anthropic Ships Claude Code Channels — Telegram + Discord
Anthropic launched Claude Code Channels as a research preview on March 20. The feature lets you connect a running Claude Code terminal session to Telegram or Discord, so you can send messages, review diffs, approve changes, and steer your coding agent from your phone. It's a two-way bridge: the agent can also push notifications and ask for approval through the messaging app.

VentureBeat called it an "OpenClaw killer," noting that Anthropic is building its own messaging-based agent interface rather than relying on third-party platforms. The current limitations: requires a terminal session open (no persistent background mode yet), only official plugins during preview, and each platform has its own API constraints.

**Why it matters:** This changes the ergonomics of AI coding. Instead of sitting at your terminal watching Claude Code work, you can message it from the couch, the gym, or a meeting. The pattern — agent-as-colleague-in-chat — is the most natural human-AI interaction model we've seen. For builders: if you're building coding agents, messaging integration just became table stakes.

## 🧑‍💻 Coding Tips

### 1) Set Up Claude Code Channels (Telegram)
```bash
# Update Claude Code to latest
claude update

# Set up Telegram bot (get token from @BotFather)
claude code --channels telegram \
  --telegram-token "YOUR_BOT_TOKEN"

# From your phone, message the bot:
# "Check the test suite for auth module"
# "Show me the diff for the last change"
# "Approve and commit"
```

Note: Research preview. Requires terminal session to stay open. Use `tmux` or `screen` for persistence.

### 2) Stripe-Style Agent PR Architecture
Based on the InfoQ deep dive, here's the pattern:

```
Slack trigger (bug/feature) 
  → Agent picks up task
  → Reads internal blueprints/docs
  → Writes code in a branch
  → Runs CI/CD (tests, lint, type check)
  → Opens PR with context
  → Human reviews (only the PR, not the process)
```

The key insight: CI/CD is the first reviewer, not a human. The agent must pass all automated checks before a human ever sees the PR. This solves the "review bottleneck" problem from the burnout reports.
