### 3. Anthropic Research: "Agentic Coding and Persistent Returns to Expertise"
Anthropic published a major research report analyzing ~400,000 Claude Code sessions from ~235,000 users over 7 months (Oct 2025–Apr 2026). Key findings: (1) In a typical session, humans make planning decisions (what to do) and Claude makes execution decisions (how to do it). (2) Every major occupation succeeds at coding tasks at nearly the same rate as software engineers. (3) But domain expertise still matters — experts get more work done per instruction and succeed more often. (4) Debugging time dropped by nearly half over 7 months. (5) Average task value rose ~25%, with work shifting toward end-to-end agentic use.

**Why it matters:** This is the best empirical data yet on how agentic coding actually works in practice. The headline finding — "persistent returns to expertise" — means AI coding amplifies your existing knowledge rather than replacing it. Non-coders can ship, but domain experts ship faster and better. For founders: invest in domain knowledge, not just AI fluency.

- [Anthropic Research](https://www.anthropic.com/research/claude-code-expertise)

## 🧪 Research Frontier

### Stanford's DeLM: Decentralized Multi-Agent Without an Orchestrator
Stanford released DeLM (Decentralized Language Model), a framework where AI agents coordinate directly via peer-to-peer communication — no central orchestrator routing every update. Results: 50% reduction in multi-agent inference costs and 10.5% improvement over SWE-bench baselines. The key insight: central orchestrators are a bottleneck and a single point of failure. Letting agents self-coordinate is cheaper and more robust.

**Alpha angle:** If you're building multi-agent systems with a central "manager" agent, DeLM suggests you might be overpaying by 2x. The peer-to-peer pattern works especially well for independent subtasks. Worth studying for anyone running multi-agent coding workflows.

## 🧑‍💻 Coding Tips

### Test Claude Agent SDK Billing Under Both Scenarios
With Anthropic's billing pause, now is the time to instrument your SDK usage so you're ready for either outcome:

```bash
# Track your Claude Agent SDK token usage per session
# Add this wrapper to your agent scripts

#!/bin/bash
# claude-metered.sh — wrapper that logs token usage
START=$(date +%s)

claude -p "$@" 2>&1 | tee /tmp/claude-session.log

END=$(date +%s)
DURATION=$((END - START))

# Extract token counts from the session log
INPUT_TOKENS=$(grep -o '"input_tokens":[0-9]*' /tmp/claude-session.log | tail -1 | cut -d: -f2)
OUTPUT_TOKENS=$(grep -o '"output_tokens":[0-9]*' /tmp/claude-session.log | tail -1 | cut -d: -f2)

echo "[USAGE] Duration: ${DURATION}s | Input: ${INPUT_TOKENS:-unknown} | Output: ${OUTPUT_TOKENS:-unknown}" >> ~/claude-usage.log

# Calculate estimated API cost (Sonnet 4.5 rates)
# Input: $3/MTok, Output: $15/MTok
if [ -n "$INPUT_TOKENS" ] && [ -n "$OUTPUT_TOKENS" ]; then
  COST=$(echo "scale=4; ($INPUT_TOKENS * 3 + $OUTPUT_TOKENS * 15) / 1000000" | bc)
  echo "[COST] Estimated API rate: \$${COST}" >> ~/claude-usage.log
fi
```

This gives you a usage log to compare flat-rate vs. metered billing when Anthropic eventually picks a direction.

## ⚙️ Workflow Upgrades

### 1. Apply DeLM's Lesson: Kill Your Central Orchestrator
Stanford's DeLM shows that peer-to-peer agent coordination beats centralized orchestration by 50% on cost. If you're running multi-agent workflows with a "manager" agent routing tasks:

```
# Instead of:
Manager Agent → Agent A, Agent B, Agent C (all comms through manager)

# Try:
Agent A ↔ Agent B ↔ Agent C (direct peer messaging)
# Let agents share context directly and decide who handles what
```

In practice, this means replacing your orchestrator prompt with shared context files that agents read/write directly — like a shared `KANBAN.md` or task queue.

### 2. Instrument Your Agent SDK Costs Before Anthropic Decides
The billing pause won't last forever. Start logging now (see Coding Tips above) so you know your actual per-session costs. When metered billing returns, you'll have data to decide whether to stay on Claude Agent SDK or switch to direct API calls.
