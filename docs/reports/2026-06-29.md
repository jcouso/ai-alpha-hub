## 🧑‍💻 Coding Tips

### Harden your coding agents against the Mozilla 0DIN attack

The clean-repo prompt injection attack works because agents trust project files. Here's how to defend:

```bash
# 1. Never auto-approve tool execution on repos you didn't create
# In Claude Code: use --dangerously-skip-permissions only on YOUR repos
# In Cursor: review all terminal commands before approval

# 2. Install agent-jackstop hardening configs
gh repo clone tenetsecurity-ai/agent-jackstop
cp agent-jackstop/.cursorrules ~/code/.cursorrules-hardened
cp agent-jackstop/CLAUDE.md ~/code/CLAUDE-hardened.md

# 3. Check for hidden prompt injection in any repo before you open it
find . -name "*.md" -o -name "*.txt" -o -name "*.json" | \
  xargs grep -li "you are\|you must\|ignore previous\|system prompt" 2>/dev/null

# 4. Run untrusted repos in a sandbox
# Docker:
docker run --rm -v $(pwd):/app -w /app node:22 npm install
# Or use GitHub Codespaces for throwaway environments
```
