## 🧪 Research Frontier

### Arbor Framework: 2.5x Better Than Claude Code and Codex on Same Compute Budget
Renmin University (Gaoling School of AI) and Microsoft Research released Arbor, an open-source optimization framework using "Hypothesis-Tree Refinement" (HTR). A coordinator agent generates hypotheses, spawns isolated executor agents (each in its own git worktree) to test them, then tree-searches through results. On ML optimization benchmarks, Arbor achieved 2.5x the average relative gain of both Codex and Claude Code on identical compute budgets.

**Alpha angle:** The architecture is what matters — tree-search over hypotheses with isolated executors is a pattern you can apply to any multi-agent system. If you're running agentic ML optimization or iterative coding, Arbor's approach of "generate hypotheses, test in parallel, prune" is likely more efficient than your current setup.

- [VentureBeat](https://venturebeat.com/orchestration/new-ai-optimization-framework-beats-claude-code-and-codex-by-2-5x-on-the-same-compute-budget)

### NVIDIA SkillSpector: 26% of Agent Skills Have Vulnerabilities
NVIDIA released SkillSpector, a static analysis tool that scans AI agent skills for malicious code, environment variable exfiltration, dynamic code execution, and prompt injection before installation. Their study found 26% of agent skills from public marketplaces contain vulnerabilities and 5% show patterns of deliberate malice. Open-source, integrates into CI pipelines.

**Alpha angle:** If you're using any agent skills marketplace (ClaWHub, npm packages for agents, MCP servers), run SkillSpector before installing. The 26% vulnerability rate is shockingly high and most teams aren't scanning at all.

- [NVIDIA/SkillSpector](https://github.com/NVIDIA/SkillSpector) | [MarkTechPost Guide](https://www.marktechpost.com/2026/06/17/nvidia-skillspector-guide-scanning-ai-skills-for-security-risks-with-static-analysis-and-sarif-reports/)

### Block's BuilderBot: Managing a Fleet of AI Coding Agents via Slack
Block (Square/Cash App) published how they built BuilderBot on top of their open-source Goose framework to manage hundreds of AI coding agents across millions of lines of code. Engineers interact with the agent fleet via Slack — assigning tasks, reviewing PRs, and coordinating across services. The system acts as a central orchestration layer for multi-agent development at enterprise scale.

**Alpha angle:** This is the most detailed public account of running AI coding agents at enterprise scale. The Slack-as-control-plane pattern is smart — it meets engineers where they already are. If you're scaling beyond 2-3 agents, Block's architecture is the reference design.

- [Developer-Tech](https://www.developer-tech.com/news/block-automates-software-development-builderbot-framework/) | [The New Stack](https://thenewstack.io/how-block-manages-its-fleet-of-ai-coding-agents-from-slack/)

## 🧑‍💻 Coding Tips

### Use Claude Code's New `/design-sync` for Brand-Consistent UIs
With the Claude Design overhaul, you can now sync your design system directly into Claude Code sessions:

```bash
# In Claude Code, sync your team's design system
/design-sync

# Claude Code now knows your brand colors, components, spacing, typography
# Generate UI that matches your design system automatically
claude "Build a dashboard page for our analytics app using our design system"

# Changes made in Claude Code sync back to Claude Design
# Changes in Claude Design sync to Claude Code
# Bidirectional — no more manual handoff
```

### Scan Agent Skills Before Installing with NVIDIA SkillSpector
26% of public agent skills have vulnerabilities. Scan before you trust:

```bash
pip install skillspector

# Scan a skill directory before installing
skillspector scan ./my-agent-skill/

# Scan a marketplace skill before download
skillspector scan --url https://clawhub.com/skills/some-skill

# Integrate into CI — block deployments with high-severity findings
skillspector scan ./skills/ --format sarif --fail-on high

# What it catches: env var exfiltration, dynamic code execution,
# prompt injection, credential harvesting, unauthorized network calls
```
