## 🧪 Research Frontier

### "Tokenomics: Quantifying Where Tokens Are Used in Agentic Software Engineering"
ArXiv paper trending on HN today (116 points). Studies how tokens are consumed across different phases of agentic coding — planning, code generation, validation, error recovery. Key finding: error recovery and context-gathering consume 3-4x more tokens than actual code generation. This has direct cost implications for anyone running AI coding agents at scale.
- [ArXiv: 2601.14470](https://arxiv.org/abs/2601.14470) | [HN](https://news.ycombinator.com/item?id=48430923)

### DeployBench: AI Agents Still Fail at the Deployment Last Mile
New benchmark testing whether AI agents can actually deploy software (not just write it). Results: agents struggle with config, environment setup, networking, and the "messy last mile" between code-complete and running-in-production. Useful calibration for anyone expecting AI agents to ship end-to-end.
- [ArXiv: 2606.05238](https://arxiv.org/abs/2606.05238)

### Anthropic: "Making Claude a Chemist"
Anthropic published their first domain-specialization research — working with synthetic, computational, and analytical chemists to make Claude better at chemistry. This is the template for how frontier models get specialized for vertical applications (pharma, materials science, etc.).
- [Anthropic Research](https://www.anthropic.com/research/making-claude-a-chemist)

### Self-Healing Agentic Orchestrators (ArXiv)
Paper on orchestration patterns where agent systems detect tool failures, classify them, and auto-recover — without human intervention. Introduces a controlled benchmark for isolating orchestration behavior from model behavior.
- [ArXiv: 2606.01416](https://arxiv.org/html/2606.01416v1)

## 🧑‍💻 Coding Tips

### Audit Your npm Dependencies for AI Tool Fakes
With the `codexui-android` supply chain attack stealing Codex tokens, and the Claude Code GH Action flaw from last week — AI tooling is now a prime attack vector.

**Quick audit for your projects:**
```bash
# Check for suspicious AI-tool-themed packages in your lockfile
grep -iE 'codex|claude|copilot|cursor|windsurf' package-lock.json | \
  grep -v 'node_modules' | \
  grep -v '@openai/codex-cli'

# Run npm audit specifically for high-severity issues
npm audit --audit-level=high

# Check for packages installed recently (last 30 days) you didn't add intentionally
npm ls --depth=0 | sort > /tmp/deps-now.txt
git show HEAD~30:package.json | jq -r '.dependencies // {} | keys[]' | sort > /tmp/deps-then.txt
diff /tmp/deps-then.txt /tmp/deps-now.txt
```

## ⚙️ Workflow Upgrades

### 1. Add a "Dependency Hygiene" Step to Your AI Coding Agent
With supply chain attacks now targeting AI tool packages specifically, add this to your `CLAUDE.md` or agent instructions:

```markdown
## Security: Before installing any npm package
1. Check the package on npmjs.com — verify publisher, download count, and GitHub link
2. Never install packages that mimic official AI tool names (codexui-*, claude-*, etc.)
3. Run `npm audit` after every `npm install`
4. Pin exact versions in production (no ^ or ~)
```

### 2. Track Your Token Spend Per Coding Phase
Inspired by the "Tokenomics" paper — most of your AI coding budget goes to error recovery and context gathering, not code generation. To optimize:

- Break large tasks into smaller, well-scoped prompts (less error recovery)
- Pre-load relevant context (file contents, API docs) instead of letting the agent explore (less context gathering)
- Use Claude Code's new Dynamic Workflows for parallel subtasks (reduces serial context accumulation)
