## 🧑‍💻 Coding Tips

### Scan for the Microsoft Package Compromise in Your Projects
With Microsoft's open source packages compromised to target AI coding agents specifically, here's how to check if you're affected:

```bash
# Check if any of your projects reference affected Microsoft packages
# The malware targets Azure and AI-dev-related packages
find ~/code -name "package-lock.json" -exec grep -l "@azure\|@microsoft" {} \;

# For each affected project, force a clean install from registry
cd /path/to/project
rm -rf node_modules package-lock.json
npm install --prefer-online

# Verify no unexpected postinstall scripts
npm ls --depth=0 | while read pkg; do
  npm info "$pkg" scripts.postinstall 2>/dev/null | grep -v "undefined" && echo "⚠️  $pkg has postinstall"
done
```

### Try Apple's Foundation Models API (Swift)
With WWDC opening Foundation Models to third-party providers, here's the new pattern:

```swift
import FoundationModels

// Works with on-device Apple model, Gemini, Claude, or OpenAI
// User chooses their preferred provider in Settings
let session = LanguageModelSession()
let response = try await session.respond(to: "Analyze this screenshot for UX issues")

// The same code works regardless of which model provider the user selected
// Apple handles routing, auth, and privacy controls
print(response.content)
```

This is a fundamentally different approach — write once, let users pick their AI provider. Similar to how Core Location abstracts GPS vs WiFi positioning.

## ⚙️ Workflow Upgrades

### 1. Implement App Intents 2.0 If You Ship iOS Apps
After WWDC, Siri AI will only call apps that implement App Intents. If your SaaS has an iOS app, this is now a top priority — without it, you're invisible in the agentic App Store. Start with your top 3 user actions and expose them as intents.

### 2. Add AI Agent Detection to Your Supply Chain Security
The Microsoft compromise specifically targets AI coding agent contexts. Add this check to your CI:

```yaml
# .github/workflows/security.yml
- name: Audit for AI-agent-targeted packages
  run: |
    npm audit --json | jq '.vulnerabilities | to_entries[] | select(.value.severity == "critical" or .value.severity == "high") | .key' || true
    # Check for unexpected postinstall scripts in dependencies
    npx lockfile-lint --type npm --path package-lock.json --allowed-hosts npm
```
