### Context Gateway: Compress Agent Context Before It Hits the LLM (84 pts HN)
Show HN launch: Context Gateway is a middleware layer that compresses agent context before sending it to the LLM, reducing token usage while preserving semantic meaning. Open source.

**Alpha angle:** As agents get longer-running and accumulate more context, compression becomes critical for cost and performance. This is complementary to prompt caching — cache the system prompt, compress the dynamic context. Worth evaluating for any agent loop processing.

## 🧑‍💻 Coding Tips

### 1) Use 1M Context Instead of RAG for Small-to-Medium Datasets
With Claude's 1M context GA, you can now skip the RAG pipeline entirely for datasets under ~500K tokens. The full-context approach is simpler and often more accurate:

```python
import anthropic

client = anthropic.Anthropic()

# Read your entire codebase / docs / dataset
with open("full_docs.txt") as f:
    full_context = f.read()  # Up to ~750K words

response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=8192,
    system="You are an expert on this codebase.",
    messages=[{
        "role": "user",
        "content": f"Here's the full documentation:\n\n{full_context}\n\nQuestion: How does the auth system work?"
    }]
)
# No chunking, no embeddings, no retrieval pipeline
# Combine with prompt-caching for 90% savings on repeated queries
```

When to still use RAG: datasets over 1M tokens, or when you need to cite exact source locations.

### 2) Enable Skip-Approval for Copilot Coding Agent in CI
If you trust your CI environment, removing the manual approval step lets Copilot's agent iterate much faster:

```yaml
# .github/workflows/copilot-agent.yml
# Add to your org/repo settings to skip approval:
# Settings > Actions > General > "Allow Copilot coding agent
#   to run workflows without approval"
#
# ⚠️ Only do this if:
# - Your CI doesn't have access to production secrets
# - Workflows are scoped to the PR branch
# - You have branch protection rules
```
