### Audit your npm/PyPI dependencies for Shai-Hulud contamination

Given the TanStack supply chain attack now affecting OpenAI and Mistral SDKs:

```bash
# Check if you installed any compromised TanStack packages
npm ls @tanstack/react-query @tanstack/router @tanstack/table 2>/dev/null | grep -E "@tanstack" | head -20

# Check for Mistral AI SDK (compromised versions on npm and PyPI)
npm ls @mistralai/mistralai 2>/dev/null
pip show mistralai 2>/dev/null

# Search for recently modified packages (last 7 days)
find node_modules -name "package.json" -newer /tmp/weekold -exec grep -l "shai-hulud\|teampcp" {} \; 2>/dev/null

# If you find anything suspicious, check the integrity
npm audit --production
pip-audit  # pip install pip-audit first
```

## ⚙️ Workflow Upgrades

### 1. Pin your dependency versions NOW
The Shai-Hulud attack exploited trust in legitimate CI/CD workflows. Lock your dependencies:
- Use `npm ci` instead of `npm install` in CI
- Enable `package-lock.json` integrity checks
- Pin exact versions in `package.json` for critical deps (remove `^` and `~`)
- Set up `socket.dev` or `snyk` for supply chain monitoring on your repos

### 2. Evaluate Grok Build if you hit Claude Code usage limits
With Anthropic tightening limits and Microsoft force-migrating to Copilot CLI, having a fallback coding agent matters. Grok Build's $300/mo SuperGrok Heavy isn't cheap, but the local-first CLI + large context window could be worth testing for large codebase work. Try at build.grok.com.
