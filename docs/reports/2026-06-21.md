## 🧪 Research Frontier

### Stanford DeLM: Decentralized Multi-Agent Coordination Cuts Costs 50%
Stanford released DeLM (Decentralized Language Model), a framework where AI agents coordinate directly via a shared verified context and asynchronous task queue — no central orchestrator needed. On SWE-bench, DeLM beats centralized baselines by 10.5% while cutting inference costs by 50%.

**Alpha angle:** If you're running multi-agent workflows (e.g., Arbor, CrewAI, or custom pipelines), the central orchestrator is your bottleneck and your biggest cost driver. DeLM's pattern — shared context + async queue — is architecturally simpler and cheaper. Worth studying if you're scaling beyond 3 agents.

- [VentureBeat](https://venturebeat.com/orchestration/stanfords-delm-cuts-multi-agent-task-costs-50-without-a-central-orchestrator)

### Anthropic Research: Domain Expertise Beats Coding Background in Agentic Coding
Anthropic published "Agentic coding and persistent returns to expertise" based on 235,000+ Claude Code users. Key finding: domain expertise (knowing the problem space) is a stronger predictor of successful agentic coding sessions than coding background. Between Oct 2025 and April 2026, code modifications dropped from 33% to 19% of all sessions — usage is shifting from "write code" to "coordinate knowledge work."

**Alpha angle:** This validates what practitioners already feel: the skill ceiling for agentic coding isn't "can you code?" — it's "do you understand the domain well enough to evaluate what the agent produces?" For agency work, this means domain experts with AI tools will outperform generalist devs.

- [Anthropic Research](https://www.anthropic.com/research/claude-code-expertise)

## 🧑‍💻 Coding Tips

### Add an AGENTS.md to Every Repo — It's Now Cross-Platform Standard
With GitHub Copilot joining Claude Code and Codex in reading AGENTS.md, this file is your universal agent instruction layer:

```markdown
# AGENTS.md

## Architecture
- Next.js 15 App Router, TypeScript strict mode
- Drizzle ORM with PostgreSQL
- All API routes in /app/api/ use edge runtime

## Conventions
- Use server actions for mutations, not API routes
- Error handling: always wrap in try/catch with typed errors
- Tests: Vitest for unit, Playwright for e2e
- Never use `any` type — use `unknown` with type guards

## Review Rules
- Flag any direct database queries outside /lib/db/
- Flag any API route missing rate limiting
- Flag hardcoded secrets or env vars in client code
```

Every AI tool that reviews your PRs, generates code, or runs tasks in your repo now reads this file automatically.

### Claude Code Auto-Mode Safety: Know What's Blocked
The June 19 update blocks destructive git commands in auto mode. If your workflow relies on these, explicitly ask:

```bash
# These are now BLOCKED in auto mode unless you ask:
git reset --hard          # blocked
git checkout -- .         # blocked  
git clean -fd             # blocked
git stash drop            # blocked
git commit --amend        # blocked if commit wasn't by agent
terraform destroy         # blocked unless you specified the stack

# To use them, be explicit in your prompt:
"Reset the working tree to HEAD and clean untracked files"
# Claude Code will then ask for confirmation before running
```
