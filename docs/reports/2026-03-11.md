## 🧑‍💻 Coding Tips

### 1) Try the GitHub Copilot SDK for embeddable agentic execution
The Copilot SDK just launched. Here's the minimal pattern to delegate a multi-step task from your app:

```typescript
import { CopilotAgent } from "@github/copilot-sdk";

const agent = new CopilotAgent({
  tools: [
    // Define your domain-specific tools
    {
      name: "query_database",
      description: "Run a read-only SQL query against the app DB",
      execute: async (params) => db.query(params.sql),
    },
    {
      name: "update_config",
      description: "Update a config value in the settings table",
      execute: async (params) => db.settings.update(params),
    },
  ],
});

// Delegate intent, not fixed steps
const result = await agent.execute({
  intent: "Check which users signed up in the last 24h and update the daily_signups config",
  constraints: {
    maxSteps: 10,
    readOnlyTools: ["query_database"], // safety constraints
    timeout: 30_000,
  },
});
```

The SDK handles planning, tool invocation, error recovery, and step-by-step execution — all within your defined constraints. Check the [cookbook examples](https://github.com/github/awesome-copilot/tree/main/cookbook/copilot-sdk/) for more patterns.

### 2) Add a senior-sign-off gate to your AI-generated code (post-Amazon outages)
Amazon's new policy: senior engineers must approve AI-assisted changes. Here's a lightweight GitHub Actions implementation:

```yaml
# .github/workflows/ai-code-review.yml
name: AI Code Review Gate
on:
  pull_request:
    types: [opened, synchronize]

jobs:
  check-ai-generated:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - name: Detect AI-generated code markers
        id: detect
        run: |
          # Check for common AI code markers in diff
          DIFF=$(git diff origin/main...HEAD)
          AI_MARKERS=$(echo "$DIFF" | grep -c -E '(Generated by|Co-authored-by:.*\+copilot|AI-assisted|claude|cursor)' || true)
          echo "ai_markers=$AI_MARKERS" >> $GITHUB_OUTPUT
      - name: Require senior review for AI code
        if: steps.detect.outputs.ai_markers > 0
        uses: actions/github-script@v7
        with:
          script: |
            await github.rest.pulls.requestReviewers({
              owner: context.repo.owner,
              repo: context.repo.repo,
              pull_number: context.issue.number,
              reviewers: ['senior-eng-1', 'senior-eng-2'], // your senior engineers
            });
            core.info('🤖 AI-generated code detected. Senior review requested.');
```

## ⚙️ Workflow Upgrades
1. **Evaluate the Copilot SDK before building more LangChain orchestration.** If you're about to add another agent chain in LangChain/CrewAI, check if the Copilot SDK's built-in planning engine handles your use case. 2. **Trim your AGENTS.md/CLAUDE.md files.** ETH Zurich research shows over-specified instruction files hurt agent performance. Keep only non-inferable details: specific coding conventions, deployment targets, domain-specific acronyms. Remove anything the agent can figure out from reading your codebase. The test: if deleting a line doesn't change the agent's behavior on your typical tasks, delete it.
