## 🧑‍💻 Coding Tips

### 1. Check Your Google API Keys for Gemini Exposure
After the CloudSEK disclosure, every developer using Google API keys needs to audit:

```bash
# Check if Generative Language API is enabled on your GCP project
gcloud services list --enabled --project=YOUR_PROJECT_ID | grep "generativelanguage"

# If enabled, ALL API keys on that project now access Gemini
# List your API keys:
gcloud services api-keys list --project=YOUR_PROJECT_ID

# Restrict keys to only the APIs they need:
gcloud services api-keys update KEY_ID \
  --api-target=service=maps.googleapis.com \
  --project=YOUR_PROJECT_ID

# Monitor for unexpected Gemini usage:
gcloud billing accounts describe ACCOUNT_ID --format="json"
```

### 2. Quick Benchmark Sanity Check for Coding Agents
Given the benchmark gaming paper, here's a practical way to evaluate coding agents on YOUR codebase instead of trusting leaderboards:

```bash
# Create a test harness for your actual project
# Pick 5 real bugs from your issue tracker and test each agent

cat > agent-eval.sh << 'EOF'
#!/bin/bash
AGENT=$1  # "claude-code" | "codex" | "cursor"
REPO_DIR=$2
ISSUE=$3

echo "=== Testing $AGENT on issue #$ISSUE ==="
echo "Start: $(date)"

# Snapshot the repo state
git -C "$REPO_DIR" stash

# Let the agent attempt the fix (manual step — observe)
echo "Run: $AGENT fix issue #$ISSUE in $REPO_DIR"
echo "Score: [pass/partial/fail] — fill in after review"
echo "Time: [minutes] — fill in"
echo "End: $(date)"
EOF
chmod +x agent-eval.sh

# Run on your own bugs. Trust your own results.
```
