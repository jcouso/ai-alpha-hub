### 3. 🚨 LiteLLM Supply Chain Attack — Check Your Deps NOW
LiteLLM versions 1.82.7 and 1.82.8 on PyPI were compromised with a multi-stage credential stealer. The malicious `litellm_init.pth` file harvests SSH keys, cloud credentials, Kubernetes configs, crypto wallets, and API keys — encrypts them and exfiltrates to an attacker-controlled domain. The attack exploited a poisoned Trivy GitHub Action to steal PyPI credentials.

**Why it matters:** LiteLLM is the go-to LLM proxy for multi-model routing — it's in a LOT of production stacks. If you use LiteLLM, audit immediately. 726 points on HN. This is the second major AI-toolchain supply chain attack in weeks.

- GitHub issue: https://github.com/BerriAI/litellm/issues/24512
- Snyk analysis: https://snyk.io/articles/poisoned-security-scanner-backdooring-litellm/
- LiteLLM security update: https://docs.litellm.ai/blog/security-update-march-2026

## 🧑‍💻 Coding Tips

### Claude Code Auto Mode — Enable It Now

Auto mode is the biggest quality-of-life improvement for Claude Code users this month. Here's how to switch:

```bash
# In Claude Code, switch to auto mode
# Press Shift+Tab to cycle through permission modes:
# 1. Normal (ask everything)
# 2. Auto (Claude decides, with safety rails)
# 3. Skip-permissions (dangerous, no rails)

# Or set it in your config:
# ~/.claude/settings.json
{
  "permissions": {
    "defaultMode": "auto",
    "blockedCommands": ["rm -rf", "DROP TABLE", "git push --force"]
  }
}
```

The key difference from `--dangerously-skip-permissions`: auto mode runs a safety classifier on each action BEFORE execution. Destructive file ops, network calls to unknown hosts, and dangerous bash patterns get flagged. Routine file reads, writes, git operations, and test runs go through silently.

### LiteLLM Emergency Audit

If you're running LiteLLM in production, run this NOW:

```bash
# Check your installed version
pip show litellm | grep Version

# If you have 1.82.7 or 1.82.8, you're compromised:
pip install litellm==1.82.6  # Downgrade to last known safe version

# Check for the malicious file
find $(python -c "import litellm; import os; print(os.path.dirname(litellm.__file__))") -name "litellm_init.pth"

# Rotate ALL credentials on affected machines:
# - SSH keys
# - AWS/GCP/Azure credentials
# - Kubernetes configs
# - API keys (OpenAI, Anthropic, etc.)
```
