## üßë‚Äçüíª Coding Tips (mandatory)

### 1) Lock down token blast radius in agentic CI (GitHub Actions)

```yaml
# .github/workflows/ci.yml
name: ci
on: [pull_request]
permissions:
  contents: read
  pull-requests: read
  issues: read
  actions: read

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm ci && npm test
```

Use write scopes only in narrowly scoped workflows that truly need them.

### 2) Enforce strict-24h freshness gates in your signal collector

```python
from datetime import datetime, timezone, timedelta
import email.utils

def in_last_24h(pubdate: str, now_utc: datetime) -> bool:
    dt = email.utils.parsedate_to_datetime(pubdate)
    if dt.tzinfo is None:
        dt = dt.replace(tzinfo=timezone.utc)
    return dt >= (now_utc - timedelta(hours=24))

# Drop stale items before ranking/reporting.
```

This prevents slow drift into 48h/7d recap mode.
