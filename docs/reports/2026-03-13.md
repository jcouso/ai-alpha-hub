**Why this matters:** This is the biggest frontend build tooling shift since Vite replaced Webpack. If you're building SaaS with React/Vue/Svelte, your CI pipeline just got dramatically faster. The Rust-ification of JavaScript tooling (Rolldown, Biome, Turbopack, SWC) is complete — the JavaScript build stack is now almost entirely Rust under the hood.

- Sources: [Vite Blog](https://vite.dev/blog/announcing-vite8), [HN — 351 pts](https://news.ycombinator.com/item?id=47360730), [ByteIota](https://byteiota.com/vite-8-0-rolldown-migration-guide-10-30x-faster-builds/)

**Alpha angle:** If transformers can execute programs natively, the line between "model" and "runtime" blurs further. This has implications for agentic AI — agents might not need to call external tools if the model itself can execute the computation. Early-stage but theoretically significant.

## 🧑‍💻 Coding Tips

### 1) Upgrade to Vite 8 — Your Builds Are About to Get 10-30x Faster
Vite 8.0 dropped yesterday with Rolldown replacing the esbuild+Rollup dual architecture. Migration is mostly seamless:

```bash
# Upgrade Vite
npm install vite@latest

# Check for breaking changes in your config
# Main thing: Rollup plugins need @rollup/plugin-* v4+
# esbuild-specific options in vite.config are deprecated

# If you have custom Rollup plugins, check compatibility:
npx vite --debug rolldown
```

Linear reported 46s → 6s on production builds. If you're on Next.js, Vite 8 + TurboPack gives you the fastest possible DX right now.

### 2) Auto-Cache Anthropic API Calls for 90% Token Savings
The new `prompt-caching` tool auto-injects cache breakpoints into your Anthropic API calls. If you're making repeated calls with similar system prompts (agent loops, batch processing, RAG pipelines):

```python
# pip install prompt-caching
from prompt_caching import CachedAnthropic

client = CachedAnthropic()  # wraps anthropic.Anthropic()

# Your existing code works unchanged — cache breakpoints
# are injected automatically based on content similarity
response = client.messages.create(
    model="claude-sonnet-4-5",
    system="Your long system prompt here...",  # cached after first call
    messages=[{"role": "user", "content": "New query"}]
)
# ~90% token savings on the system prompt portion
```
