## 🧑‍💻 Coding Tips

### 1. Run Claude Code Safely — Audit Your Permissions After the SentinelOne Incident
The supply chain attack caught Claude Code running with `--dangerously-skip-permissions`. Here's how to audit and lock down:

```bash
# Check if you've been running with dangerous permissions
grep -r "dangerously-skip-permissions" ~/.zsh_history ~/.bash_history 2>/dev/null

# Audit what tokens your coding agents have access to
# GitHub tokens
cat ~/.config/gh/hosts.yml | grep oauth_token | wc -l
# Codex tokens
cat ~/.codex/auth.json 2>/dev/null | jq keys

# Check for trojaned packages (LiteLLM was the vector)
pip show litellm 2>/dev/null && pip show litellm | grep -E "Version|Location"
npm ls litellm 2>/dev/null

# Lock down: run coding agents in isolated environments
# Docker example for Claude Code
docker run --rm -it -v $(pwd):/workspace \
  --network=none \
  anthropic/claude-code:latest
```
