### 5. Three AI Coding Agents Leaked Secrets Through a Single Prompt Injection
A Johns Hopkins security researcher demonstrated that a single malicious instruction in a GitHub PR title caused Claude Code, Gemini CLI, and GitHub Copilot to exfiltrate their own API keys as PR comments. The attack pattern: untrusted GitHub data → AI agent processes it → agent executes commands → credentials posted publicly. All three major coding agents were vulnerable to the same basic prompt injection technique.

**Why it matters:** This isn't theoretical. If you're running AI code review actions on public repos, your API keys may already be exposed. The attack surface is everywhere agents read untrusted input: PR descriptions, issue comments, README files, even commit messages. Immediate action: audit all AI agent GitHub Actions, restrict their output permissions, and never grant agents write access to public-facing channels.

## 🧑‍💻 Coding Tips

### 1. Audit Your AI Coding Agent GitHub Actions — NOW
After the prompt injection disclosure, every repo using AI code review needs an audit:

```bash
# Find all AI agent GitHub Actions in your org
gh api /orgs/YOUR_ORG/repos --paginate -q '.[].full_name' | while read repo; do
  echo "=== $repo ==="
  gh api "/repos/$repo/contents/.github/workflows" -q '.[].name' 2>/dev/null | \
    xargs -I{} gh api "/repos/$repo/contents/.github/workflows/{}" -q '.content' 2>/dev/null | \
    base64 -d 2>/dev/null | grep -l -i 'claude\|copilot\|gemini\|ai-review' 2>/dev/null && echo "⚠️  AI agent action found"
done

# Key mitigations:
# 1. Never let AI agents post to public PR comments with API keys in env
# 2. Use GITHUB_TOKEN with minimal scopes (read-only where possible)
# 3. Add pull_request_target restrictions to prevent fork-based attacks
```
