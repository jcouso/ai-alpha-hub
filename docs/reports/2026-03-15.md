### 3) Glassworm Supply-Chain Attack: 151+ Repos, npm, VS Code
Security researchers revealed that the Glassworm threat actor compromised 151+ GitHub repos, npm packages, and VS Code extensions using invisible Unicode characters to hide malicious payloads. The attack steals tokens, credentials, and secrets using blockchain-based exfiltration. Targets include repos from Wasmer, anomalyco (OpenCode/SST), and Reworm. Researchers suspect LLMs generated the packages at scale.

**Why this matters:** This is the first major confirmed case of AI-generated supply-chain attacks at scale. The invisible Unicode technique means standard code review won't catch it — the malicious payload renders as zero-width whitespace in every editor and terminal. March 2026 Patch Tuesday also addressed 6 vulnerabilities in Microsoft's AI agent orchestration layer, making this the first month where AI agent security became a formal patching category.

### Microsoft AgentRx: Systematic Debugging for AI Agents
Microsoft Research published AgentRx — a framework that pinpoints the first unrecoverable failure step in AI agent runs by synthesizing executable constraints from tool schemas and domain policies. It logs evidence-backed violations step-by-step, giving developers a debugger-like experience for agent workflows.

**Alpha angle:** Agent debugging is going to be as important as application debugging. AgentRx's approach (automated constraint synthesis from tool schemas) is the right abstraction — instead of reading agent logs, you define what "correct behavior" looks like and the framework finds where the agent deviated. If you're building agent systems, this is essential infrastructure.

- Sources: [Microsoft Research](https://www.microsoft.com/en-us/research/blog/systematic-debugging-for-ai-agents-introducing-the-agentrx-framework/)

## 🧑‍💻 Coding Tips

### 1) Audit Your Dependencies for Glassworm
The invisible Unicode attack won't show up in standard code review. Run this to check for suspicious Unicode in your dependency tree:

```bash
# Quick check: scan node_modules for Private Use Area chars
grep -rP '[\x{FE00}-\x{FE0F}\x{E0100}-\x{E01EF}]' node_modules/ 2>/dev/null | head -20

# Better: use Aikido's scanner or semgrep rules
# npm install -g @aikido/cli && aikido scan
```

Affected repos include Wasmer, anomalyco (OpenCode/SST), and Reworm. If you depend on these, audit immediately.

### 2) Try the gstack Pattern for Claude Code
Instead of freeform prompting, give Claude Code structured roles:

```markdown
<!-- .claude/commands/review.md -->
## Role: Senior Code Reviewer
## Scope: This PR only
## Output: Markdown with severity labels

Review the current PR for:
1. Logic errors and edge cases
2. Security vulnerabilities
3. Performance regressions
4. API contract violations

For each finding, provide:
- Severity (critical/major/minor)
- File + line
- What's wrong
- Suggested fix
```

This is the core pattern from gstack and The Agency. Role + scope + explicit deliverables = dramatically better agent output.
