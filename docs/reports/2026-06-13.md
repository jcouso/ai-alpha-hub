### 2. "Agentjacking" — Fake Bug Reports Hijack AI Coding Agents
Tenet Security published research on a new attack class called "agentjacking" that exploits the intersection of Sentry's open event ingestion and MCP tool integration. An attacker crafts a fake Sentry error event containing malicious instructions. When an AI coding agent (Claude Code, Copilot, etc.) reads the Sentry error via MCP, it trusts the content and executes arbitrary code on the developer's machine. No malware, no stolen credentials, no breach of the target. Disclosed to Sentry on June 3, 2026.

**Why it matters:** If you use an AI coding agent connected to Sentry via MCP, you're exposed right now. The attack exploits implicit trust between tools — the exact pattern MCP was designed to enable. This is a wake-up call for anyone with MCP integrations: treat all external data sources as untrusted input, even error trackers.

- [The Hacker News](https://thehackernews.com/2026/06/agentjacking-attack-tricks-ai-coding.html) | [Tenet Security (original research)](https://tenetsecurity.ai/blog/agentjacking-coding-agents-with-fake-sentry-errors/) | [Cloud Security Alliance](https://labs.cloudsecurityalliance.org/research/csa-research-note-agentjacking-mcp-sentry-injection-20260612/) | [The Next Web](https://thenextweb.com/news/agentjacking-ai-coding-agents-sentry)

## 🧪 Research Frontier

### Agentjacking: A New Attack Primitive for Agentic Systems
Tenet Security's agentjacking research isn't just a vulnerability disclosure — it defines a new attack primitive. The core insight: AI agents that read from external data sources (error trackers, databases, APIs) via MCP tools treat that data with implicit trust. An attacker doesn't need to compromise the agent or the tool — just inject data into a source the agent reads. This generalizes beyond Sentry to any MCP-connected data source. The Cloud Security Alliance published a formal research note analyzing the implications for agentic architectures.
- [Cloud Security Alliance research note](https://labs.cloudsecurityalliance.org/research/csa-research-note-agentjacking-mcp-sentry-injection-20260612/) | [Tenet Security original](https://tenetsecurity.ai/blog/agentjacking-coding-agents-with-fake-sentry-errors/)

### AA-AgentPerf: First Benchmark for Agent Infrastructure
Artificial Analysis's AgentPerf is the first attempt to standardize how we measure hardware capacity for agentic workloads. Unlike traditional LLM benchmarks (tokens/second), AgentPerf measures concurrent agent support under realistic coding trajectories. The benchmark normalizes per-accelerator and per-megawatt, making it useful for infrastructure planning. Initial results show Blackwell's architectural advantages (larger HBM, faster interconnects) translate directly into agent density.
- [NVIDIA Developer Blog](https://developer.nvidia.com/blog/nvidia-achieves-leading-agentic-coding-performance-on-first-agentic-ai-benchmark/)

### Compression Is Intelligence — 3Blue1Brown Series
3Blue1Brown released a multi-part series on entropy and compression, starting with "Reinventing Entropy" (June 7) and culminating in "Measuring the Entropy of English" (June 12). The series builds from Shannon's information theory to modern language models, connecting data compression directly to intelligence — the argument that prediction (what LLMs do) and compression are mathematically equivalent. Beautifully produced and surprisingly relevant to understanding why LLMs work.
- Part 1: [Reinventing Entropy](https://www.youtube.com/watch?v=l6DKRf-fAAM) | Part 3: "Measuring the entropy of English" (June 12)

### Nature: General-Purpose LLMs Outperform Specialized Clinical AI
A Nature paper shows that general-purpose large language models now outperform specialized clinical AI tools on medical benchmarks. This challenges the "fine-tune a vertical model" approach — suggesting that scale and general capability may matter more than domain-specific training for many professional tasks.

## 🧑‍💻 Coding Tips

### Audit Your MCP Integrations for Agentjacking Exposure
The agentjacking research shows any MCP tool connected to an external data source is a potential attack vector. Here's how to check your exposure:

```bash
# Find all MCP server configs in your Claude Code / Cursor setup
find ~/.config -name "*.json" -exec grep -l "mcp" {} \; 2>/dev/null
cat ~/.config/claude/claude_desktop_config.json | jq '.mcpServers | keys[]' 2>/dev/null

# Check specifically for Sentry MCP integration
grep -r "sentry" ~/.config/claude/ ~/.cursor/ 2>/dev/null

# If you have Sentry MCP connected, validate your DSN isn't public
# Attackers can inject fake events if they know your Sentry DSN
# Check your client-side code for exposed DSNs:
grep -r "DSN" --include="*.ts" --include="*.js" --include="*.env" . | grep -i sentry
```

**Mitigation:** If you must use Sentry MCP, add input validation to your MCP server that strips any instruction-like content from error messages before passing to the agent. Better yet: treat all MCP-sourced data as untrusted user input.

### Pin Your OpenAI Model IDs Before They Break
GPT-5.2 just disappeared without advance notice. Check your production code:

```bash
# Find all OpenAI model references in your codebase
grep -rn "gpt-5\.2\|gpt-4\.5\|o3-" --include="*.ts" --include="*.py" --include="*.env" .

# Quick migration: replace deprecated models
# gpt-5.2-instant → gpt-5.5
# gpt-5.2-thinking → gpt-5.5-thinking  
# gpt-4.5 → gpt-5.5 (sunset June 27)
# o3 → o3-pro or gpt-5.5-thinking (sunset Aug 26)

# Pro tip: abstract your model selection
# In your .env or config:
# OPENAI_MODEL=gpt-5.5
# Then in code: model = process.env.OPENAI_MODEL || "gpt-5.5"
```

## ⚙️ Workflow Upgrades

### 1. Add Multi-Provider Fallback After the Fable 5 Shutdown
The government just demonstrated that frontier model access can be revoked overnight. If you're building for international users, you need provider redundancy. Add a fallback chain:

```typescript
// Model fallback chain — never depend on one provider
const MODELS = [
  { provider: 'anthropic', model: 'claude-sonnet-4-5', priority: 1 },
  { provider: 'openai', model: 'gpt-5.5', priority: 2 },
  { provider: 'google', model: 'gemini-3-pro', priority: 3 },
  { provider: 'mistral', model: 'mistral-large', priority: 4 }, // sovereign option
];

// If your primary provider goes dark, auto-route to next available
async function resilientCall(prompt: string) {
  for (const m of MODELS) {
    try { return await callModel(m.provider, m.model, prompt); }
    catch { continue; }
  }
  throw new Error('All providers unavailable');
}
```

### 2. Sandbox Your MCP Integrations
After agentjacking, treat MCP connections like you'd treat user input — never trust them implicitly. Add a `.claude/settings.json` rule to require confirmation for any code execution triggered by MCP data:

```json
{
  "permissions": {
    "mcp": {
      "sentry": { "requireConfirmation": true },
      "database": { "requireConfirmation": true }
    }
  }
}
```
