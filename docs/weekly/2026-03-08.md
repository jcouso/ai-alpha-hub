### AI Alpha Weekly — 2026-03-08

### 🧭 Week Narrative
A semana marcou uma virada de “mais um modelo” para “stack de execução real”. OpenAI empurrou GPT‑5.3 Instant e GPT‑5.4 em sequência curta, com foco explícito em trabalho profissional (planilhas, documentos, computer-use e contexto 1M). Isso não é só benchmark theater: é disputa direta por fluxo de trabalho diário.

Ao mesmo tempo, segurança virou feature central de produto. OpenAI lançou Codex Security com validação orientada a contexto; Anthropic mostrou resultados de red teaming no Firefox com 22 vulnerabilidades reportadas. O recado é claro: agente sem segurança embutida vai morrer em produção, especialmente com os incidentes da semana (deleção de ambientes, guias falsos de instalação, ataques por prompt/config).

No lado builder, o jogo abriu: Cursor Automations, Copilot mais agentic, stacks open source subindo (OpenSandbox, deer-flow, Superset, gws). O padrão vencedor agora é: agente + sandbox + memória + governança + observabilidade. Quem continuar só no “chat + patch” vai ficar para trás rápido.

### 🔬 Top Papers of the Week
**SWE-CI: Evaluating Agent Capabilities in Maintaining Codebases via Continuous Integration** (arXiv:2603.03823)  
Propõe benchmark de manutenção contínua (100 tarefas, média de 233 dias de evolução e 71 commits), saindo do bugfix pontual.  
Ângulo prático: mede o que importa para produto vivo — regressão, iteração longa e confiabilidade de agente ao longo do tempo.  
https://arxiv.org/abs/2603.03823

**BeyondSWE: Can Current Code Agent Survive Beyond Single-Repo Bug Fixing?** (arXiv:2603.03194)  
Mostra teto <45% para modelos de ponta em cenários mais reais (cross-repo, migração, geração full-repo).  
Ângulo prático: valida que “agente resolve bug isolado” ainda não equivale a “agente entrega roadmap”.  
https://arxiv.org/abs/2603.03194

**SWE-rebench V2: Language-Agnostic SWE Task Collection at Scale** (arXiv:2602.23866)  
Dataset/pipeline para treinar agentes em 32k+ tarefas, 20 linguagens e 3,600+ repositórios com ambiente reproduzível.  
Ângulo prático: base real para RL/eval interno multi-stack (não só Python/JS) em times SaaS heterogêneos.  
https://arxiv.org/abs/2602.23866

**CUDA Agent: Large-Scale Agentic RL for High-Performance CUDA Kernel Generation** (arXiv:2602.24286)  
Sistema agentic RL para otimização de kernels, reportando ganhos fortes em KernelBench e superando modelos proprietários no split difícil.  
Ângulo prático: reforça tese de agentes especialistas por domínio com ambiente e recompensa próprios.  
https://arxiv.org/abs/2602.24286

**SkillNet: Create, Evaluate, and Connect AI Skills** (arXiv:2603.04448)  
Infraestrutura para criar/avaliar/conectar skills com ontologia e critérios de segurança/executabilidade/manutenibilidade/custo.  
Ângulo prático: casa diretamente com estratégia de skill libraries para reduzir retrabalho de prompt e padronizar qualidade.  
https://arxiv.org/abs/2603.04448

**Building AI Coding Agents for the Terminal** (arXiv:2603.05344)  
Discute arquitetura terminal-first com separação planner/executor, compaction de contexto e memória automática.  
Ângulo prático: blueprint aplicável para evoluir teus agentes sem explodir contexto ou risco operacional.  
https://arxiv.org/abs/2603.05344

### 🧪 Test This Week
**1) Subir um sandbox real para execução de agente (OpenSandbox)**  
```bash
uv pip install opensandbox-server
opensandbox-server init-config ~/.sandbox.toml --example docker
opensandbox-server
```  
**Expected outcome:** tu ganha um runtime isolado para testar automações com menos risco de “agente mexer onde não deve”.

**2) Colocar Google Workspace no teu stack de automação (gws CLI)**  
```bash
npm install -g @googleworkspace/cli
gws auth setup
gws auth login
gws drive files list --params '{"pageSize": 5}'
```  
**Expected outcome:** integra Drive/Gmail/Calendar com saída JSON, pronto para plugar em agentes comerciais/operacionais.

**3) Rodar o “agent loop mínimo” para nivelar arquitetura do time**  
```bash
git clone https://github.com/shareAI-lab/learn-claude-code
cd learn-claude-code
pip install -r requirements.txt
cp .env.example .env
python agents/s01_agent_loop.py
```  
**Expected outcome:** benchmark conceitual rápido para comparar teu fluxo atual com uma implementação limpa de loop + tools.

**4) Testar orquestração de múltiplos coding agents com isolamento**  
```bash
git clone https://github.com/superset-sh/superset.git
cd superset
cp .env.example .env
echo 'SKIP_ENV_VALIDATION=1' >> .env
bun install && bun run dev
```  
**Expected outcome:** valida se teu ciclo “Claude implementa → Codex revisa → apply” fica mais rápido com worktrees paralelos e painel único.
